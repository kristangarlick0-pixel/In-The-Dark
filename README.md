# In The Dark

## Project Description
**In The Dark** is a hero shooter centered around the classic battle of good versus evil. Players compete in objective-based game modes, such as escorting a convoy, while choosing to fight as either heroes or villains. This interface showcases the **villain selection screen**, where players can browse the available villains, read their backstories and abilities, view the teammates' selected characters, and see the current game mode and map before the match begins.

## Intended User or Player
This interface is designed for players who enjoy competitive multiplayer games and the choice of playing as either a hero or a villain. The game captures the fantasy of becoming a powerful character capable of changing the outcome of a battle. It also appeals to strategic players, as understanding how different abilities interact is essential for teamwork and success. Fast-paced action and team coordination make the game enjoyable for players who thrive in competitive environments.

## User / Player Goal
The player's goal is to compare the available villains, learn about their abilities and roles, coordinate with teammates, and confidently select a character before the match begins.

## Inspiration Interfaces
- **Overwatch** inspired the clean layout, organized character cards, and intuitive hero selection interface.
- **Marvel Rivals** inspired the bold character presentation, vibrant visuals, and overall modern art direction.

## Design Choices
- **Layout:** A responsive grid of character cards divided into three class sections, alongside a featured panel displaying the currently selected villain.
- **Color:** Dark blacks, grays, and deep reds create an ominous villain-themed atmosphere while allowing characters and interface elements to stand out.
- **Typography:** Large white and red headings provide strong visual contrast, while clean body text keeps information readable.
- **Visual Hierarchy:** The selected villain and page title attract attention first, followed by the character grid and supporting interface elements.
- **Feedback States:** Character cards, navigation links, and buttons change color when hovered over to indicate that they are interactive.
- **Responsive Design:** The character grid adjusts from multiple columns on larger screens to a single-column layout on smaller devices.
